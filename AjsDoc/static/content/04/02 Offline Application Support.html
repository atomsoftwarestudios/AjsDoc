Offline Application Support

Offline application support was one of main requiremets the framework has to fullfil. The support is currently based on the Application Cache browser feature, what is going to be deprecated and removed from web browsers. Once this will happen the framework will switch to Service Workers technology, what in the end will have no impact to existing applications usig AJS as this change will be transparent to them (of course only if application will be properly designed and will follow AJS best practices).

The Application Cache is based on the manifest file with details describing what files should be cached and used in offline mode and another details. For details see Using the application cache article at the MDN site.

Resources which were cahced to the Application Cache are never updated until the appcache.manifest file is modified. The post-build tool is included in the tools directory and updates the appcache.manifest file automatically when:

  • index.html file is modified or
  • ajs.js file is modified or
  • ajs.boot.config.js file is modified or
  • favicons or appplication manifests file are modified

Only files mentioned above are stored in the application cache. Other files (such as application resources) are managed by AJS Resource Manager if the application does not use any different approach.

Once the application gets offline there is fallback section defined in the cache manifest file which redirects all requests to index.html page (better said to the root of the web - /). All resources requested in the offline mode are then handled as the index.html file (even if request for image is made). In order to be possible to recognize the applicaion is oflline and cached resource should be returned (instead of index.html) the resource loader inspects the resource content returned by the browser and is checking for <!--offline--> tag in first 50 bytes of loaded resource. If the offline marker is found the appropriate the resource loader returns 304 - Not modified instead of the resource loaded and resource manager uses the resource from the cache. So this means the index.html file has to contain this offline marker the in order the application work properly. The index.html should look like the following example: 

<!DOCTYPE html>
<!--offline-->
<html lang="en" manifest="/cache.manifest">
    <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1; minimum-scale=1; maximum-scale=1; user-scalable=no">
        <meta name="apple-mobile-web-app-capable" content="yes" />
        <meta name="theme-color" content="#ffffff">
        <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
        <link rel="icon" type="image/png" href="/favicon-32x32.png" sizes="32x32">
        <link rel="icon" type="image/png" href="/favicon-16x16.png" sizes="16x16">
        <link rel="manifest" href="/manifest.json">
        <link rel="mask-icon" href="/safari-pinned-tab.svg" color="#5bbad5">
        <title>AjsDoc - The Ajs Framework Documentation</title>
        <script src="/js/ajs.boot.config.js"></script>
        <script src="/js/ajs.js"></script>
    </head>
    <body>
    </body>
</html></code>
