Introduction


the Atom JavaScript Frontend Framework




 !!! PLEASE NOTE THE FRAMEWORK IS STILL UNDER DEVELOPMENT AND THE DOCUMENTATION IS SUBJECT TO CHANGE !!! 


AJS (pronounce /aɪs/) is JavaScript Single Page Application framework written in TypeScript focused to the rapid web browser application development. The resulting code is still pure JavaScript (current target is ES5) so its features and possibilities were taken to the account during the development.

As it was required to support older versions of web browsers (mainly ie 10+, because Edge is still not fully completed) there are no features like Promises or Service Workers used at this time but it should not take long to update the framework to support these technologies.

The code was designed to be as simple as possible so no special features of the TypeScript language, like decorators, were not used. Also, there is no need to externally inject anything to anywhere or load external libraries unless you specifically need it.

The data binding is not and will not be supported directly by the framework as there is no direct support for it in the JavaScript (it is still necessary to anotate the data with getters and setters what would not be that bad as need of converting arrays to object structures in order to be possible to monitor its changes - hopefully once we will get indexers as they are implemented in the C#)

The ouptut code is not optimized as author wish, because although the TypeScript compiler is great it is still not ideal, but hopefully we can live with it.

Set of build tools will grow as there will be need for it (ie. plans for the preprocessor allowing IFDEFS exists, but it will take a while to be developed). It will be mainly focused on the Visual Studio and MS Build so if you would like to have also other enviroments you have to contribute ;)

Features

Browser Support

  • AJS Framework is designed to fully support major Web Browsers
IE 10+ Edge Chrome 4+ Firefox 2+ Safari 3.1+ Opera 9+

Offline Application Support

  • Support for full or partial offline Web Applications
  • The Application cache browser feature is used to support offline applications
  • Application resources can be managed and cached using the framework itself
  • Includes support for cache and appliaction updates with last recent versions

Boot Process Control

  • Simple initial index.html page
  • AJS Framework Configuration
  • Custom Application Configuration
  • Configurable resource loading

Navigation & Routing & Redirection management

  • Managed navigation with notifications
  • Configurable path-to-path redirection
  • Configurable path-to-view component routing

Resource Management & Caching

  • Resources are loaded from the server using AJAX requests
  • Resources can be cached in one of available storages – Local, Session or Memory
  • Automatic script evaluation
  • Callback system to notify the caller the resource/s was/were loaded.

MVVM Architecture

  • Model View View Model architecture
  • Innovative approach to Veiw with the Visual and View Components
  • View Component (ViewModel) drives the flow through the application
  • Models fully controllable by developers

Visual Component, View Component, View Model, View

  • Inspired by React approach to the state management
  • View Components are completely based on the logic-less templates called Visual Components
  • Event based notification about user actions through the tree of components
  • Automatic DOM rendering and updating based on template and the view component state
